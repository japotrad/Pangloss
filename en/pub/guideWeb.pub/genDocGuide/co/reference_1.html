Supported Features

Feature

In a glossary, the actual information is embedded in features. Some Features apply only to Entries. Others apply only to Terms. And some Features may apply both to Entries and Terms. A Feature has a name and a value. The name of the feature can be viewed as the type of data attached to the Entry or the Term. It corresponds to the data categories that can be found at www.isocat.org.

The Features Pangloss supports are listed below. Features applying to Entries are marked with the icon. Features applying to Terms are marked with the icon.

Entries and Terms Features

Feature name

Definition

Example

context

Example of use. A text which illustrates the use of a designation.

Here is a context sentence illustrating the term samples gallery: The program may contain sample source code or programs, which illustrate programming techniques. These samples are found in the samples gallery.

customerSubset

An identifier indicating that a Term is associated with a specific customer.

If the IBM company uses the above term samples gallery, then the IBM customerSubset shall be attached to this term.

definition

A representation of a concept by a descriptive statement which serves to differentiate it from related concepts.

The concept of progressive lens may be defined as follows: Corrective lenses used in eyeglasses to correct presbyopia and other disorders of accommodation, characterised by a gradient of increasing lens power, added to the wearer's correction for the other refractive errors.

entryIdentifier

An alphanumeric string that

serves as the unique identifier of an Entry.

These codes are usually generated automatically by the application. Example: eid-Oracle-67

languageID

Identifier of the language that is included in the Entry. Typically, this is the language of the definition of a concept.

You should use normalized (e.g. ISO) codes. You may use a locale instead of a language code. Examples: ENG, EN, en-US.

objectLanguage

Identifier of the language for which information is provided, i.e. the language of the term feature.

You should use normalized (e.g. ISO) codes. You may use a locale instead of a language code. Examples: ENG, EN, en-US.

projectSubset

An identifier assigned to a specific project indicating that it is associated with a Term.

These codes are usually generated by project managers. Example: TD-

13

term

A verbal designation of a general concept in a specific subject field. This feature is the most important feature of a Term. It can consist of single words or be composed of multiword strings. The distinguishing characteristic of a term is that it is assigned to a single concept, as opposed to a phraseological unit, which combines more than one concept in a lexicalized fashion to express complex situations.

Quality assurance system is a term, whereas satisfy quality requirements is a phraseological unit, specifically a collocation.
